The Surface Shift

The CLI was the adoption surface.

The raw notes preserve a practical story: coding harnesses, executable interfaces, and a stack of internal CLIs became the work surface people actually used. That matters because agents get more useful when they live where files, repos, commands, review, and execution already meet. [1][2][4]

25CLIs described as built in 10 weeks
30-40PRs/day noted against the CLI operating load
4engineers noted as supporting the CLI team

The point is not that everyone should live in a terminal. The point is that the work substrate matters. If an agent can see the repo, call tools, inspect history, propose changes, and run checks in one place, the GUI-vs-CLI debate is too small.

The Core Problem

The bottleneck moves from generation to absorption.

Once agents make candidate work cheap, the scarce capacity becomes validation: review, security, architecture alignment, behavioral verification, regression detection, and human taste. [2][3]

accelerated

lagging

This is the velocity gap: writing can scale faster than the organization can prove, review, and safely absorb the work.

The Operating Model

Research, Gates, Sweeps is not a checklist.

It is a loop. Research maps the system before acting. Gates block known failure modes before promotion. Sweeps find recurring rot that one PR or one review cannot see. [1][2][3]

HMC Translation

Do not copy the terminal. Copy the substrate logic.

The HMC lesson is not "make everyone use a CLI." It is that powerful agentic surfaces only work when the underlying capabilities are bounded, observable, policy-aware, and reviewable. [4][5]

Wrong abstraction

Agent inside a generic app with broad tools, unclear authority, hidden side effects, and review after the fact.

Better abstraction

Agent inside a governed work substrate: typed capabilities, proposal-first workflows, policy gates, evals, traces, and deterministic execution.

The strategic question is where HMC work already lives: files, tickets, data, approvals, system access, review, audit, and execution. The agent surface should meet that substrate instead of pretending work starts in a blank chat box.
