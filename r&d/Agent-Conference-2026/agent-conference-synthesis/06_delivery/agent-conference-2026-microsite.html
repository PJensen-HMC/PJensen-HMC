Executive read

Build the control plane, not the magic trick.

The conference did not make the case for unbounded agent rollout. It made the case for a governed execution architecture: identity, delegated authority, capability surfaces, context provenance, validation loops, observability, deterministic execution boundaries, and human review.

Read the argument Pick the first pilot
What changed

Agents are moving from chat objects to non-human actors that inspect, propose, call tools, generate artifacts, and pressure enterprise systems.

What it means

The design unit is not the model. The design unit is the governed capability: typed input, known side effect, policy, audit, replay, and recovery.

What HMC should do

Pilot one bounded, measurable toil workflow. Use it to build reusable control-plane muscle before broad autonomy spreads.

Core argument

Do not pilot an agent. Pilot a governed surface.

A successful pilot should prove whether HMC can expose a narrow, useful, auditable capability to non-human actors without creating invisible authority or unbounded operational load.

01 / Translation Vendor demos are the surface layer.

The durable pattern is below the demo: auth, policy, evals, observability, data quality, tool contracts, infrastructure cost, and recoverability.

02 / Architecture Capabilities beat endpoints.

Raw API exposure gives agents too much ambiguity. A capability should have a business name, owner, contract, side-effect class, policy, audit event, and rollback path.

03 / Operating model Validation is the throughput limit.

AI generates candidate work first. The enterprise bottleneck becomes review, gates, sweeps, trace inspection, recovery, and human taste.

Control-plane map

The architecture is the safety model.

Click each layer. The point is not novelty; it is where failures become observable, reversible, and accountable.

Adoption sequence

Small blast radius. Real governance. Reusable patterns.

Name the work

Pick a concrete toil workflow. Avoid vague “agent platform” framing.

Freeze the boundary

Read-only or proposal-first until ownership, audit, and review are proven.

Encode the surface

Define typed input, output, side effects, permissions, trace fields, and recovery.

Measure the loop

Track time saved, review burden, false positives, drift, rework, and incidents.

Promote the pattern

Only generalize the control surface after the pilot proves useful and governable.

Pilot recommendation

Favor boring, measurable toil.

The first pilot should be dull enough to govern and useful enough to matter. It should create evidence, not theater.

Decision rules

What to avoid. What to build.

Avoid: broad agent access to raw systems.

It will look fast until identity, permissions, prompt injection, state mutation, audit gaps, and infrastructure load become operational problems.

Build: named capabilities with policy.

Expose narrow actions with typed contracts, known side effects, actor rules, trace events, review gates, and rollback or compensation behavior.

Avoid: demo-first success metrics.

“It worked in the room” is not a production claim. Measure end-to-end throughput, review burden, failures, recoverability, and user trust.

Build: gates, sweeps, and replay.

Use repeatable evals, variance checks, policy gates, recurring sweeps, and event history so the organization can inspect what happened after the fact.

Field evidence explorer

Method note

The capture method was itself a finding.

The conference was captured as a human-agent research pair. The human sensed the room and chose signal. The agent normalized, compared, extracted, reconciled, and preserved caveats.

Human layer

Room selection, credibility sensing, emotional salience, photos, audio, raw reactions, skepticism, and deciding what mattered under time pressure.

Agent layer

Schedule reconciliation, extraction, claim ledgers, caveat preservation, synthesis, source indexing, and turning a firehose into an inspectable artifact.

Source spine

This site is a presentation layer.

The durable record should remain the markdown packet, extracted claims, normalized ledgers, and raw evidence tree.
