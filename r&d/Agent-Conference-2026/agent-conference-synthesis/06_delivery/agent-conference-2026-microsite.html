Agent Conference 2026 / HMC Executive Microsite

Build the control plane.

The conference did not validate a simple "agents everywhere" roadmap. It surfaced a production architecture: non-human actors need explicit identity, governed capabilities, provenance-aware context, validation, observability, infrastructure controls, and human-review boundaries. [1][2]

Method Signal

Human-agent capture was part of the finding.

This was not only a memo about agents. It was produced through a human-agent research loop: the human drank from the conference firehose; the agent carried normalization, extraction, comparison, caveat preservation, and synthesis. [1][4][5]

Human role

Choose rooms, sense credibility, notice energy, record audio, photograph slides, and mark what felt actionable in motion.

Agent role

Reconcile notes and schedules, index sources, extract claims, preserve caveats, and turn high-noise capture into inspectable synthesis.

HMC Thesis

Do not assume existing access surfaces are agent-ready.

APIs, wrappers, and integration layers are often application plumbing. That is not sufficient for non-human actors. The strategic move is a design standard for governed capability surfaces. [1][2]

Capabilitynamed action, not just endpoint
Contracttyped input, typed output
Side Effectsknown impact and owner
Policypermissions and actor constraints
Auditevent, trace, replay path
Recoveryrollback or compensation

The near-term work is to introduce the concept, test it in one bounded pilot, and decide which existing system-access surfaces could be refactored or wrapped into this model over time.

What Carries Forward

Seven things to remember.

Actors, not users

Separate subject, actor, delegation, tool, and system of record.

Capabilities

Expose narrow, typed, policy-aware actions instead of raw system access.

Context

Treat retrieval as infrastructure with provenance, freshness, permissions, and cost.

Validation

Generation accelerates first. Review and recovery become the bottleneck.

Domain rules

The scarce resource is not tokens. It is encoded judgment.

Infrastructure load

Humans pause. Agents do not. Model agent-shaped traffic separately.

Use-case ethics

Start with toil and reliability, not opaque authority or containment.

Human center

Agents carry volume; humans retain standards, taste, and authority.

Operating Controls

Validation is the new throughput constraint.

Across the coding-agent, validation, and infrastructure material, the pattern is the same: AI accelerates candidate work first. The organization has to scale review, gates, sweeps, replay, evals, and incident response around that new volume. [1][2][3]

small reviewable diffs provenance test selection behavioral verification security checks architecture checks coverage ratchets migration guards CODEOWNERS replay drift sweeps human taste

This is not process theater. It is how the enterprise absorbs AI velocity without surrendering control.

Low-Regret HMC Moves

Start bounded. Build reusable control.

Pick one toil pilot

Named owner, measurable success criteria, small blast radius, proposal-first or read-only starting mode.

Define a capability registry

Owner, contract, side-effect class, actor policy, audit fields, rollback path, and criticality level.

Set identity and gateway policy

Subject/actor separation, delegated authority, scoped permissions, prompt-injection response, and destructive-action handling.

Make context provenance real

Source ID, version, permission, freshness, index version, and output references.

Turn evals into guardrails

Variance checks, repeatability checks, replay, gates, sweeps, and incident-response loops.

Capture domain rules

Reusable skills, domain-rule libraries, encoded decision procedures, and capability policies.

Caveats

Keep the evidence honest.

Vendor and product material is field signal, not procurement recommendation.

NVIDIA CLI adoption details are note-backed, not audio-backed in the current corpus.

The "1B tables" scale anecdote remains attribution-caveated.

OCR-backed slide wording should be checked before quotation.

The HMC capability-surface recommendation is an HMC interpretation of the conference pattern, not a direct conference claim.

HMC should not rush to trust agents. HMC should shape the world agents are allowed to inhabit.
